## Changes from v3

**Adopted (folded in)**:
- Q1: `User.token` → `User.token_digest` for cookie security. Cookie carries plaintext, DB stores hash.
- Q2: `Mobile::BaseController#ensure_user!` removed; `requires_user!` filter only on actions that need it (onboarding, auth). Marketing pages don't create User rows.
- Q3: `User#normalise_email_address` callback (downcase + strip before save) — works correctly with deterministic encryption.
- Q4: `numericality` validation on `times_divorced` / `times_widowed` (≥ 0).
- Q6: Migration ordering note rewritten as "atomic deploy."
- Q7: `User#first_incomplete_path` and `welcome_complete?`/`location_complete?`/etc. inlined explicitly.
- Q8: Server-side login confirmation as backup to Stimulus warning.
- Q11: Encryption-at-rest test.
- Q12: Cookie cutover test at registration.
- Elon E5: Wave 2 packaged as 5 atomic commits, not 12.

**Adjusted with Q2 in mind**:
- Will-from-day-one preserved. With Q2 in place ("only create User on meaningful action"), Wills are created for engaged users only — not for marketing-page visitors. Acceptable.

**Explicitly rejected**:
- Elon E2 ("don't materialize anything until welcome step"): too disruptive vs the Q2 partial fix. Q2 captures most of the benefit (no User on marketing pages); going further requires rebuilding the attribution capture mechanism. Defer to a future plan if bot traffic actually becomes a concern.
- Elon E3 (drop Will-from-day-one): the user explicitly confirmed Will-from-day-one. With Q2, the cost is negligible.
- Q10 (filter person-picker dropdown): keeping the long list. Epic 4 person-merge feature handles cleanup.
- Q13 (cleanup job load test): deferred to PRE_LAUNCH_TODO.
- Q14, Q15, Q16, Q17 nits: deferred.

## Note to AI agent

Forbidden:
- No service objects (`app/services/` is off limits).
- No form objects, presenter objects, "policy" objects.
- No new gems beyond what's already in the Gemfile.
- No premature extraction of helpers, concerns, or shared modules until at least three duplicates exist.
- No defensive code for cases that can't happen.
- No rescuing exceptions you don't have a specific recovery for.
- Multi-row write logic lives on the relevant Active Record model as fat-model methods, not in service-object classes.
- No default scopes (DHH dislikes them; explicit > implicit).

```ruby
class User < ApplicationRecord
  has_secure_password validations: false

  has_many :sessions, dependent: :destroy
  has_many :api_sessions, dependent: :destroy
  has_many :devices, dependent: :destroy

  has_many :people, dependent: :destroy
  belongs_to :will_maker_person, class_name: "Person", optional: true
  has_one :spouse_person, -> { where(relationship_kind: "spouse") },
          class_name: "Person", inverse_of: :user

  has_many :wills, dependent: :destroy
  has_one :active_will, -> { where(status: "active") }, class_name: "Will"
  has_one :draft_will, -> { where(status: "draft") }, class_name: "Will"

  encrypts :email_address, deterministic: true

  validates :email_address,
    uniqueness: { case_sensitive: false },
    presence: true,
    on: :register
  validates :password,
    length: { minimum: 12 },
    format: { with: /\A(?=.*[a-zA-Z])(?=.*[0-9]).*\z/, message: "must contain letters and numbers" },
    on: :register

  # Q3: normalise email before encryption to match deterministic encryption uniqueness
  before_save :normalise_email_address

  # Will-from-day-one
  after_create :create_initial_will

  attr_accessor :token  # Q1: transient — DB stores token_digest
  before_create :assign_token

  # === State predicates ===

  def anonymous?
    email_address.nil?
  end

  def registered?
    email_address.present?
  end

  # === Auth ===

  def self.authenticate_by(email_address:, password:)
    return nil if email_address.blank? || password.blank?
    normalised = email_address.downcase.strip
    user = where.not(email_address: nil).find_by(email_address: normalised)
    return nil if user.nil? || user.password_digest.blank?
    user.authenticate(password) ? user : nil
  end

  def self.find_by_cookie_token(plaintext)
    return nil if plaintext.blank?
    digest = Digest::SHA256.hexdigest(plaintext)
    find_by(token_digest: digest, email_address: nil)
  end

  # === Onboarding step predicates (Q7 inlined) ===

  def first_incomplete_path
    return Rails.application.routes.url_helpers.mobile_onboarding_welcome_path unless welcome_complete?
    return Rails.application.routes.url_helpers.mobile_onboarding_location_path unless location_complete?
    return Rails.application.routes.url_helpers.mobile_onboarding_family_path unless family_complete?
    return Rails.application.routes.url_helpers.mobile_onboarding_extended_family_path unless extended_family_complete?
    Rails.application.routes.url_helpers.mobile_onboarding_wrap_up_path
  end

  def welcome_complete?
    will_maker_person.present? &&
      will_maker_person.first_name.present? &&
      will_maker_person.last_name.present? &&
      will_maker_person.date_of_birth.present?
  end

  def location_complete?
    will_maker_person.present? &&
      will_maker_person.country_of_residence.present? &&
      will_maker_person.nationality.present? &&
      will_maker_person.domiciled_in_uk.present? &&
      will_maker_person.currently_resident_in_uk.present?
  end

  def family_complete?
    current_step.in?(%w[extended_family wrap_up]) || completed_at.present?
  end

  def extended_family_complete?
    current_step == "wrap_up" || completed_at.present?
  end

  # === Lifecycle ===

  def inactive_for_more_than?(duration)
    last_seen_at && last_seen_at < duration.ago
  end

  def touch_last_seen!
    update_column(:last_seen_at, Time.current)
  end

  # === Onboarding step writes (fat-model methods) ===
  # apply_welcome_step!, apply_location_step!, apply_family_step!,
  # apply_extended_family_step!, apply_partner!, apply_children!, apply_co_parent!,
  # destroy_existing_partnership!, destroy_orphaned_co_parents!
  # Implementation identical to v2 [§W2.5]; just operates on `self.people` not `onboarding_session.people`.

  private

  def normalise_email_address
    self.email_address = email_address.downcase.strip if email_address.present?
  end

  def assign_token
    self.token = SecureRandom.hex(32)
    self.token_digest = Digest::SHA256.hexdigest(token)
  end

  def create_initial_will
    wills.create!(version: 1, status: "draft")
  end
end
```

#### 2B. BaseController (Q2 fix folded in)

```ruby
class Mobile::BaseController < ApplicationController
  helper_method :current_user

  before_action :touch_user_if_present

  private

  def current_user
    @current_user ||= find_current_user
  end

  def find_current_user
    if user_id = session[:user_id]
      registered = User.find_by(id: user_id)
      return registered if registered
      session.delete(:user_id)  # stale; clear
    end

    if cookie_token = cookies.signed[:user_token]
      anonymous = User.find_by_cookie_token(cookie_token)
      return anonymous if anonymous && !stale?(anonymous)
      cleanup_stale_user!(anonymous, cookie_token) if anonymous
    end

    nil
  end

  # Only call from controllers that need a User (onboarding, auth).
  # Marketing pages (intro, video-intro, risk-questionnaire) do NOT create Users.
  def requires_user!
    @current_user = current_user || create_anonymous_user!
  end

  def create_anonymous_user!
    user = User.create!
    cookies.signed[:user_token] = {
      value: user.token,  # transient plaintext
      expires: 3.hours.from_now,
      httponly: true,
      same_site: :lax,
      secure: Rails.env.production?
    }
    user
  end

  def stale?(user)
    user.last_seen_at.present? && user.last_seen_at < 3.hours.ago
  end

  def cleanup_stale_user!(stale_user, cookie_token)
    stale_user.destroy
    cookies.delete :user_token
    @current_user = nil
  end

  def touch_user_if_present
    current_user&.touch_last_seen! if current_user&.anonymous?
  end

  def authenticate_user!
    return if current_user&.registered?
    redirect_to mobile_login_path
  end
end
```

#### 4A. SessionsController

```ruby
class Mobile::SessionsController < Mobile::BaseController
  rate_limit to: 10, within: 3.minutes, only: :create

  def new
    # Q8: server-side check as backup for Stimulus warning
    @has_onboarding_data = current_user&.anonymous? && current_user.current_step.present?
  end

  def create
    user = User.authenticate_by(
      email_address: params.require(:email_address),
      password: params.require(:password)
    )

    if user
      destroy_anonymous_user!
      session[:user_id] = user.id
      cookies.delete :user_token
      redirect_to mobile_dashboard_path
    else
      redirect_to mobile_login_path, alert: "Invalid email or password"
    end
  end

  def destroy
    session[:user_id] = nil
    redirect_to mobile_intro_path
  end

  private

  def destroy_anonymous_user!
    return unless current_user&.anonymous?
    current_user.destroy!
    @current_user = nil
    cookies.delete :user_token
  end
end
```

#### 4B. AuthController

```ruby
class Mobile::AuthController < Mobile::BaseController
  before_action :requires_user!

  def secure_account
    return redirect_to mobile_dashboard_path if current_user.registered?
    @user = current_user
  end

  def create
    return redirect_to mobile_dashboard_path if current_user.registered?

    user = current_user
    user.email_address = params.require(:email_address)
    user.password = params.require(:password)

    if user.save(context: :register)
      session[:user_id] = user.id
      cookies.delete :user_token
      redirect_to mobile_dashboard_path
    else
      @user = user
      render :secure_account, status: :unprocessable_entity
    end
  end
end
```
